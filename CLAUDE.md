# NVO Automation — project instructions for Claude

This repo generates Leap Finance's country-specific study-abroad ad videos
("NVOs") via HyperFrames (HTML/CSS/GSAP → MP4). Full docs live in two places —
read them before doing anything non-trivial here:

- **`.claude/skills/nvo-generator/SKILL.md`** — the actual workflow (intake,
  footage sourcing, compose, render, review). This is a Claude Code **skill**:
  it auto-triggers on `/nvo-generator` or natural asks like "make an NVO for
  Ireland" — nothing needs installing for that to work, it's auto-discovered
  the moment this folder is open in Claude Code.
- **`NVO_STYLE_BIBLE.md`** — the design rationale behind every visual/beat
  decision, reverse-engineered from 9 real campaigns. Read this before making
  any styling call the skill file doesn't already cover.

## One-time setup (only needed before actually rendering something)

1. `npm install`
2. Copy `.env.example` → `.env`, fill in `PEXELS_API_KEY` and `GEMINI_API_KEY`.
3. This machine's default `node` may be under the required v22+ — if so,
   prefix commands with `export PATH="/opt/homebrew/opt/node/bin:$PATH"`
   (adjust to wherever Homebrew installed a newer node).

## Hard rules — do not reintroduce these (each was a real bug found in practice)

- **Never share assets or render output across campaigns.** Every campaign is
  fully self-contained: `nvo-template/assets/<campaign>/` (shots + logos
  together) and `nvo-template/compositions/<campaign>/` (generated output),
  scoped by the spec's own filename. A shared pool for either one previously
  caused campaigns to silently mix up or overwrite each other.
- **Never ask the user to place a file inside this project's folders
  themselves.** For a named university/company logo, fetch the real one
  yourself first (WebSearch/WebFetch its Wikipedia infobox logo, then
  `curl` it down — verified working, no user action needed at all). Only
  fall back to grabbing it off their clipboard or asking where it already
  lives (Desktop, Downloads, `inbox/`) when no good web match exists — see
  SKILL.md's Architecture section for the exact commands and caveats of each.
- **Never edit anything under `nvo-template/compositions/`** — always
  generated by `scripts/compose.mjs`, overwritten on every run.
- **Ask intake questions one at a time**, not batched — this was explicitly
  requested after batching several fields into one message was rejected.
- **`nvo-template/CLAUDE.md` is generic `hyperframes init` boilerplate that
  does NOT describe this project's actual workflow** (it mentions `npm run
  dev`, which this project never uses) — always defer to
  `.claude/skills/nvo-generator/SKILL.md` instead when working inside
  `nvo-template/`.
